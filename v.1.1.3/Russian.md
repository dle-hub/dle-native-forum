Возможности

· Полноценная форумная система: категории, темы, сообщения.
· AJAX-действия без перезагрузки страницы: лайки, подписки, редактирование.
· Иерархический список категорий в стиле XenForo.
· Двойной SEO-движок: поддержка DLE 18 (.htaccess) и DLE 19+ (DLEUrl).
· Префиксы тем (Prefix) и система тегов (Tags).
· Sticky OP: закрепляет первое сообщение на всех страницах темы.
· Отслеживание прочитанного/непрочитанного (для участников и гостей).
· Продвинутая панель управления: drag-and-drop категорий, массовая модерация, редактирование тем, журнал действий.
· Интеграция с DLE Conversations (личные сообщения работают через стандартную систему DLE).
· Система званий, жалобы, запрещённые слова, загрузка файлов и многое другое.

---

Требования

· Версия DLE: 18.0 и выше (рекомендуется 19+).
· PHP: 7.4+ (рекомендуется 8.1+).
· MySQL: 5.7+ или MariaDB 10.0+.
· Плагин: не изменяет оригинальные файлы DLE, конфликты исключены.

---

Структура файлов

```
/engine/forum/          → Ядро форума (классы, экшены)
/engine/inc/forum.php   → Панель управления
/engine/modules/        → Интеграция с профилем и боковыми блоками
/templates/Default/forum/ → Пример файлов темы (TPL + CSS)
/language/              → Языковые файлы (forum.lng.php)
```

При установке в систему добавляются только необходимые файлы. Ни один системный файл DLE не изменяется.

Установка (Пошагово)

⚠️ Важно: Простая загрузка XML-файла недостаточна. Выполните все шаги по порядку.

1. Установка XML

1. Панель управления DLE → Плагины → Загрузить плагин.
2. Выберите файл dle-native-forum-ultimate-core-v113.xml и загрузите его.
3. Активируйте плагин.
4. Плагин автоматически:
   · создаст необходимые таблицы и столбцы в базе данных;
   · добавит SEO-правила в .htaccess (для DLE 18).

2. Загрузка языкового файла

1. Возьмите файл language/Russian/forum.lng.php из репозитория (или папку вашего языка).
2. Загрузите его на сервер по пути /language/Russian/forum.lng.php (или другой язык).
3. Если файл уже существует — перезапишите.
4. Без языкового файла сообщения об ошибках и панель управления будут отображаться на английском (fallback).

3. Загрузка файлов темы (TPL)

· Возьмите все файлы из папки templates/Default/forum/ в репозитории.
· Скопируйте их в папку вашей темы: /templates/ВАША_ТЕМА/forum/.
· Пример: /templates/Default/forum/ → /templates/MyTheme/forum/
· Список шаблонов:
  ```
  main.tpl           → Главная страница
  category.tpl       → Категория и результаты поиска
  topic.tpl          → Просмотр темы
  new_topic.tpl      → Создание новой темы
  notifications.tpl  → Уведомления
  pm.tpl             → Личные сообщения (теперь перенаправляет в стандартную систему DLE)
  user_posts.tpl     → Сообщения пользователя
  forum_block.tpl    → Боковой блок (последние темы)
  ```
· Вы можете редактировать файлы и адаптировать их под свой дизайн.

4. Обновление CSS и JS

· Поместите forum.css из репозитория рядом с CSS вашей темы или импортируйте его.
· Не пропускайте новые CSS-классы: в новой версии добавлены .df-post-sticky, .df-sticky-label, .df-prefix, .df-tags-bar.
· Для работы TinyMCE-редактора на страницах форума убедитесь, что в <head> вашей темы присутствуют теги {js-files} и {AJAX}.

5. Очистка кэша

1. Панель управления DLE → Настройки → Очистить кэш.
2. Для очистки кэша форума:
      ?mod=forum&action=clear_cache (кнопка «Очистить кэш» в панели форума) или вручную удалите файлы, начинающиеся с forum_, из папки /engine/cache/.

6. Обновление статистики

· Сразу после установки нажмите кнопку «Пересчитать статистику» (?mod=forum → раздел «Обслуживание»).
· Это исправит счётчики категорий и пользователей.

---

После установки

· SEO не работает: для DLE 18 убедитесь, что файл .htaccess доступен для записи. Для DLE 19+ дополнительных действий не требуется.
· Группа администраторов автоматически получает полный доступ. Настройте «Права категорий» для других групп.
· Создайте префиксы и теги в панели управления (?mod=forum&action=prefixes, ?mod=forum&action=tags).
· Убедитесь, что Sticky OP включён: ?mod=forum&action=settings → значение sticky_op должно быть 1.

Для разработчиков тем

Список всех тегов, доступных в .tpl-файлах, приведён в разделе Справочник тегов.
Важные моменты:

· {posts_list} содержит HTML сообщений; структура формируется в topic.php. При необходимости вы можете редактировать PHP-код.
· Блочные теги [logged]...[/logged] и [not-logged]...[/not-logged] позволяют разделять контент для гостей и авторизованных.
· {editor} выводит поле для ответа (WYSIWYG или BBCode).
· Шаблон панели управления (engine/inc/forum.php) независим от темы.

Частые ошибки и их решение

🐞 "Ошибка MySQL", таблицы не созданы

· На MySQL 5.7 конструкция ADD COLUMN IF NOT EXISTS не работает. Отключите и снова включите плагин — это повторно выполнит upgrade SQL.
· Если проблема сохраняется, создайте необходимые таблицы вручную (см. секцию mysqlupgrade внутри XML-файла).

🐞 Вёрстка сломана, новые стили не отображаются

· Вы не обновили CSS. Добавьте в тему актуальный forum.css из репозитория.
· Вы загрузили не все TPL-файлы. Проверьте папку /templates/ВАША_ТЕМА/forum/.

🐞 "Sticky OP не работает", первое сообщение на других страницах отсутствует

· Убедитесь, что в настройках sticky_op = 1.
· Очистите кэш форума.
· Sticky OP отображается только со 2-й страницы (на первой первое сообщение и так видно).

🐞 Клик на последнее сообщение ведёт в начало темы (битая ссылка)

· Эта ошибка была в v1.1.2. Если вы обновлялись, проверьте, что файлы engine/forum/actions/main.php и category.php обновились.
· Если в вашем TPL используется тег {last_post_link}, проблемы быть не должно.

🐞 Панель управления / форум не переведены (русский, английский и т.д.)

· Ваш языковой файл устарел. В новой версии добавлены десятки ключей. Скачайте актуальный forum.lng.php из репозитория и загрузите в свою языковую папку.
· Если перевода для каких-то ключей нет, функция отобразит резервный текст на английском. Вы можете помочь с переводом!

🐞 Кнопка «Редактировать» переносит тему в редактор быстрого ответа

· Это штатное поведение. Редактирование сообщения происходит на той же странице через редактор ответа.
· В панели управления доступно редактирование в WYSIWYG-редакторе: ?mod=forum&action=edit_topic&id=X.

Обновление (v1.1.2 → v1.1.3)

1. Загрузите новый XML-файл в панель (он перезапишет старую версию).
2. Отключите и снова включите плагин. Это добавит новые таблицы и столбцы.
3. Замените TPL-файлы и CSS на актуальные из репозитория.
4. Обновите языковой файл (добавлено много новых ключей).
5. Очистите кэш.
6. Пересчитайте статистику.

Часто задаваемые вопросы (FAQ)

· Можно ли использовать плагин, загрузив только XML?
    Нет. Необходимо также загрузить TPL- и CSS-файлы в папку вашей темы.
· SEO не работает на DLE 18.
    Убедитесь, что файл .htaccess доступен для записи, и переактивируйте плагин.
· Почему счётчик ботов всегда равен 0?
    DLE не хранит ботов в таблице _users (группы 6/7). Счётчик ботов пока равен нулю.
· Русский языковой файл неполный?
    Проверьте актуальную версию в репозитории или помогите с переводом!
